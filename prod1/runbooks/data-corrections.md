# Runbook: Data Corrections (Production)

## Когда использовать
- Некорректная сводка из-за исторических артефактов.
- Нужно массово проставить теги/номера кошельков.

## Правила
- Любая правка данных только в production-контуре.
- Перед массовой правкой: зафиксировать источник (CSV/таблица) и правило.
- После правки: верификация выборкой + UI-проверка.

## Кейс: self-transfer
Симптом: завышенная сводка.
Причина: строки `from == to == wallet.address` в ledger.
Решение:
1. Очистка исторических строк.
2. Защитная фильтрация self-transfer в API (`transfers`, `summary`, `balances`).

## Кейс: номера кошельков по строкам CSV
Правило: номер строки с валидным адресом = `walletNumber` и одноимённый tag.
Пустые строки без адреса — игнорировать.
